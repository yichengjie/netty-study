1.可以把Channel看着传入(入站)或者传出(出站)数据的载体。因此，它可以被打开或者被关闭，连接或者断开连接。
2.netty是一个网络编程框架，所以事件是按照它们与入站或出站数据流的相关性进行分类的。
3.可能由入站数据或者相关的状态更改而触发的时间含括：
  3.1 连接已被激活或者连接失活；
  3.2 数据读取
  3.3 用户事件
  3.4 错误事件
4.出站事件是未来将会触发的摸个动作的操作结果，这些动作包括:
  4.1 打开或者关闭到远程节点的连接
  4.2 将数据写到或者冲刷到套接字
5.事件和EventLoop
  EventLoop本身只由一个线程驱动，其处理了一个Channel的所有I/O事件，并且在该EventLoop的整个生命周期内都不会改变。
  
6.所有netty服务器都需要以下两部分
  6.1 至少一个ChannelHandler,该组件实现了服务器对从客户端接收的数据的处理，即它的业务逻辑
  6.2 引导，这是配置服务器的启动代码。至少，它会将服务器绑定到它要监听连接请求的端口上。
  
7.从一个客户端应用程序的角度来看，如果事件的运动方向是从客户端到服务器，那么我们称这些事件为出站的，反之则称为入站。
  7.1 数据的出站运动（即正在被写的数据）在概念上也是一样的。
